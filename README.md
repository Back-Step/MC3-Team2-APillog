MC3-Team2 repository

# Git convention
## Branch 생성 규칙
* [작업태그]/이슈번호   
ex) [FEAT]/#12 

## Issue 생성 규칙
* 기본적으로는 이슈 템플릿을 사용하여 작성

## Commit Message 규칙
### 제목
* 50자 이내
* 단어 형태로 종결
* [작업태그] #이슈번호 - 작업 사항
### 본문
* 80자 이내
* 단어 형태로 종결
* 작업 사항 설명  

ex) [FEAT] #12 - 00추가
* 00과 관련된 뷰 추가

## PR(Pull Request) 규칙
### 제목
* [작업태그] 작업 사항 축약  
ex) [FEAT] 회원가입 뷰 추가
### 본문
* PR 생성 후, 오른쪽 하단의 Development에서 연관된 이슈를 연결하기
---
|*작업태그*|*내용*|
|:---|:---|
|**FEAT**|새로운 기능 추가/ 일부 코드 추가 / 일부 코드 수정(리팩토링과 구분)|
|**FIX**|버그 수정|
|**DOCS**|문서 수정|
|**TEST**|테스트 코드 추가|
|**REFACTOR**|코드 리팩토링| 
|**STYLE**|코드 의미에 영향을 주지 않는 변경사항|
|**CHORE**|빌드 부분 혹은 패키지 매니저 수정사항|
